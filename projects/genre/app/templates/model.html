Models
One architecture, delivered as a tiered set (proposal §3b). Each FORGE reproduction ships between one and three trained models so the platform can showcase differences between models, not just one model's uncertainty.
Floor = Model 1. Only the faithful reproduction is a hard deliverable. Model 2 is the goal wherever the timeline allows; Model 3 is opportunistic. The Bayesian layer is last-layer Laplace (LLLA) for the interactive path with HMC as ground truth — the deterministic backbone stays in PyTorch, only the final layer goes Bayesian.

MODEL 2 Author-optimized (RRM) target
The same architecture, with the configuration chosen by the reliability-aware sweep — scored on accuracy, fold-stability, and posterior uncertainty together, not accuracy alone (Appendix C).
Robust Ranking Metric, over 3-fold CV: $$v = \left(\,1-\bar a,\ \tfrac{\sigma_{\text{fold}}}{\sigma_{\max}},\ \tfrac{U_{\text{mean}}}{U_{\max}}\,\right), \qquad \mathrm{RRM} = 1 - \lVert v \rVert_2$$ equal-weight centroid $w=(\tfrac13,\tfrac13,\tfrac13)$; selection maximizes RRM.
sweep
~20 configs × 3-fold CV (the mk5l / mk5m setup)
reporting
RRM is selection-only, never reported — results use standard metrics (accuracy, P/R/F1, ROC-AUC, ECE)
axes
inaccuracy 1−ā · instability σ_fold/σ_max · posterior uncertainty U_mean/U_max
Source: FORGE proposal Appendix C (Exploratory Reliability-Aware Selection Metrics).
